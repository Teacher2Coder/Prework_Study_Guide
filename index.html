HTML

  • The head element contains information about the webpage that is not seen by the user
    • "title" defines a title for the webpage
    • "style" defines the CSS style associated with the page
    • "base" defines the base URL for the webpage
    • "link" connects external resource to an HTML document
    • "meta" defines ceertain meta data such as:
      • "charset" which specifies the characters used in an HTML document
      • "content" which specifies values associated with a named attribute
      • "http-equiv" provides an HTTP header for information and values
      • "viewport" describes the visable area for the webpage
    • "script" embeds client-side data scripts in HTML
  • The body element contains the visable content shown to the user
    • Heading tags start with an h and a number with a number increased as the level of the heading increases
    • Paragraph tags ("p") allows text to be written in paragraph format
    • List tags include ordered/numbered lists ("ol") or unordered/bulleted lists ("ul") with each item listed as "li"
    • Images are inserted with "img" commands
    • Breaks in texts are added with a "br /" command
    • Hyperlinks or Anchors are inserted with the "a" command with "href" pointing to where you want the link to take you

CSS

  • Inline and Internal CSS
    • Inline: Applies directly to a specific HTML tag in a specific line
    • Internal: Contains CSS rules for an entire webpage included in the head section
    • Not very flexable or easily changed in a uniform way
  • External CSS
    • Keeps CSS data on a seperate file that can be linked to the HTML file
    • Selectors: Pick the HTML element that you want to stylize
    • Declarations: A sepcific style that you want to apply to an element
    • Property: A category of style that you want to apply to the element
    • Value: The visual change that you want applied to the element
    • Link external CSS files in the head of the HTML file

Git

  • Important terms for GitHub
    • "Repository": a project on GitHub typically stored in a folder
    • "Branch": a workspace for creating change that doesn't affect the main site
    • "Markdown": the way content files are formatted in GitHub
    • "Commit Changes": saving a new version of a file that you have been working on that only affects the branch you are working on
    • "Pull Request": Asking for changes made to the code to be added to the main branch
    • "Merge": after a PR is approved, the code in added to the main branch and deployed to the webpage
    • "Issue": a way to report problems or to assign new tasks
  • Git commands in CLI
    • "git status": show modified files in working directory, staged for your next commit
    • "git add -a": add a file as it looks now to your next commit
    • "git pull origin main": checks to see if current branch is up to date
    • "git push origin [branch name]: pushes changes in current branch to GitHub
    • "git branch": provides a list of your branches, a * will appear next to the currently active branch
    • "git branch [branch name]": create a new branch at the current commit
    • "git checkout [branch name]": switch to another branch and check it out into your working directory
    • "git checkout -b [name of new branch]": create a new branch off another branch
    • "git commit -m [descriptive message]: commits changes and adds a message describing the changes made
  • The GitHub Workflow
    1. Create a branch
    2. Make changes to that branch
    3. Create a pull request
    4. Address review comments
    5. Merge pull request

Javascript

  • Variables allow you to store datapoints for later arguments
    • "var" adds a variable
    • "=" assigns the variable to a datapoint
  • Control Flow is the order the computer will perform certain operations
    • If code is out of order, it will not function properly
    • Control Flow can be interupted with Conditional Statements and For Loops
  • Arrays allow us to store several datapoints similarly to a variable
  • Functions are a set of instructions that the computer follows to perform a task
    • Two steps: Define the function, then call the function
    • Defining the function involves naming the function and assigning code to it
    • Calling the function occurs at the bottom and is placed in a work order
