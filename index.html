HTML

  • The head element contains information about the webpage that is not seen by the user
    • "title" defines a title for the webpage
    • "style" defines the CSS style associated with the page
    • "base" defines the base URL for the webpage
    • "link" connects external resource to an HTML document
    • "meta" defines ceertain meta data such as:
      • "charset" which specifies the characters used in an HTML document
      • "content" which specifies values associated with a named attribute
      • "http-equiv" provides an HTTP header for information and values
      • "viewport" describes the visable area for the webpage
    • "script" embeds client-side data scripts in HTML
  • The body element contains the visable content shown to the user
    • Heading tags start with an h and a number with a number increased as the level of the heading increases
    • Paragraph tags ("p") allows text to be written in paragraph format
    • List tags include ordered/numbered lists ("ol") or unordered/bulleted lists("ul") with each item listed as "li"
    • Images are inserted with "img" commands
    • Breaks in texts are added with a "br /" command
    • Hyperlinks or Anchors are inserted with the "a" command with "href" pointing to where you want the link to take you

Git

  • Important terms for GitHub
    • "Repository": a project on GitHub typically stored in a folder
    • "Branch": a workspace for creating change that doesn't affect the main site
    • "Markdown": the way content files are formatted in GitHub
    • "Commit Changes": saving a new version of a file that you have been working on that only affects the branch you are working on
    • "Pull Request": Asking for changes made to the code to be added to the main branch
    • "Merge": after a PR is approved, the code in added to the main branch and deployed to the webpage
    • "Issue": a way to report problems or to assign new tasks
  • The GitHub Workflow
    1. Create a branch
    2. Make changes to that branch
    3. Create a pull request
    4. Address review comments
    5. Merge pull request
